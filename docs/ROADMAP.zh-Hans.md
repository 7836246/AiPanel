# AiPanel 路线图

AiPanel 的目标不是复刻传统服务器面板，而是做一个本地运行的 AI 运维客户端。路线图会优先验证安全执行闭环，再扩展图形界面和应用部署。

## M0：项目定义

- [x] 明确产品定位：本地 AI 运维客户端；
- [x] 明确差异点：服务器零常驻、SSH 优先、AI 先规划再执行；
- [x] 准备 README、中文文档、许可证和品牌素材；
- [x] 准备 GitHub 发布所需社区文件。

## M1：CLI MVP

目标：先用命令行验证核心链路。

- [ ] 本地配置目录；
- [ ] 服务器连接配置；
- [ ] SSH 连通性测试；
- [ ] 系统基础信息采集；
- [ ] 只读服务器体检；
- [ ] 命令输出采集；
- [ ] 本地任务记录。

## M2：AI 任务规划

目标：让 AI 生成可审计计划，而不是直接执行命令。

- [ ] 自然语言转结构化任务；
- [ ] 任务步骤拆分；
- [ ] 命令风险分级；
- [ ] 用户确认流程；
- [ ] 执行结果总结；
- [ ] 失败原因分析。

## M3：安全执行器

目标：建立可信执行边界。

- [ ] 只读模式；
- [ ] 危险命令识别；
- [ ] 二次确认；
- [ ] 命令白名单；
- [ ] 执行超时和中断；
- [ ] 审计日志；
- [ ] 敏感信息脱敏。

## M4：桌面客户端

目标：提供适合普通用户的本地客户端体验。

- [ ] 服务器列表；
- [ ] 任务对话界面；
- [ ] 执行计划预览；
- [ ] 实时命令输出；
- [ ] 任务历史；
- [ ] 本地密钥和配置管理。

## M5：应用部署能力

目标：覆盖高频轻量服务器场景。

- [ ] Docker 安装；
- [ ] Docker Compose 部署；
- [ ] Nginx / Caddy 反向代理；
- [ ] HTTPS 证书配置；
- [ ] 应用健康检查；
- [ ] 常见应用模板，例如 Uptime Kuma、n8n、WordPress、PostgreSQL、Redis。

## 暂不做

- 不做云端托管控制台；
- 不默认安装服务器常驻 agent；
- 不默认开放新的公网管理入口；
- 不直接执行未审查的 AI 命令；
- 不把用户服务器凭据上传到第三方服务。
