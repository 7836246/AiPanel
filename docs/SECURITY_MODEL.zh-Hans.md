# AiPanel 安全模型

AiPanel 的核心风险来自 AI 生成命令、远程服务器执行、凭据管理和日志处理。因此安全模型必须先于功能扩展建立。

## 基本原则

- AI 只能生成计划，不能绕过确认直接执行；
- 默认优先只读诊断；
- 写操作必须可见、可审查、可确认；
- 高风险命令必须二次确认；
- 凭据只保存在本地；
- 审计记录必须可回看；
- 不默认安装服务器常驻 agent；
- 不默认开放新的公网管理入口。

## 风险等级

### Low

只读检查，不改变服务器状态。

示例：

- 查看系统版本；
- 查看 CPU、内存、磁盘；
- 查看端口监听；
- 查看服务状态；
- 读取少量日志。

### Medium

会改变服务状态，但通常可恢复。

示例：

- 安装软件包；
- 拉取 Docker 镜像；
- 启动或重启服务；
- 写入新的应用配置；
- 新增反向代理配置。

### High

可能造成数据丢失、服务中断或安全边界变化。

示例：

- 删除文件；
- 清空日志或数据库；
- 修改防火墙规则；
- 修改 SSH 配置；
- 覆盖已有 Nginx 配置；
- 递归改权限；
- 格式化磁盘。

### Blocked

默认禁止，除非未来提供非常明确的专家模式。

示例：

- `rm -rf /`；
- 格式化系统盘；
- 清空生产数据库；
- 关闭 SSH 登录且没有回滚方案；
- 执行来源不明的远程脚本。

## 命令审查

命令审查应识别：

- 删除类命令：`rm`、`find -delete`；
- 权限类命令：`chmod -R`、`chown -R`；
- 磁盘类命令：`mkfs`、`fdisk`、`parted`；
- 网络类命令：`iptables`、`ufw`、`firewalld`；
- 服务类命令：`systemctl restart`、`docker compose down`；
- 远程脚本：`curl | sh`、`wget | bash`；
- 数据库破坏性语句：`DROP`、`TRUNCATE`。

## 只读模式

只读模式下只允许检查类命令。

允许：

- `uname`；
- `uptime`；
- `df`；
- `free`；
- `ss`；
- `ps`；
- `systemctl status`；
- `docker ps`；
- `journalctl` 限量读取；
- 读取配置文件。

不允许：

- 安装软件；
- 修改配置；
- 重启服务；
- 删除文件；
- 修改防火墙；
- 写入数据库。

## 凭据管理

AiPanel 应将敏感信息保存在本地安全存储中。

敏感信息包括：

- SSH 私钥；
- SSH 密码；
- sudo 密码；
- API Key；
- 数据库密码；
- 服务器连接配置中的敏感字段。

要求：

- 不写入 Git 仓库；
- 不写入明文日志；
- 不发送给 AI，除非用户明确授权且确有必要；
- 展示时默认脱敏。

## 日志和脱敏

发送给 AI 或写入审计记录前，应尽量脱敏：

- IP 地址；
- Token；
- 密码；
- Cookie；
- Authorization 头；
- 私钥内容；
- 数据库连接串；
- 云厂商 AK/SK。

## 审计记录

每次执行应记录：

- 用户原始意图；
- AI 计划；
- 风险等级；
- 用户确认；
- 实际执行命令；
- 执行时间；
- 退出码；
- 脱敏输出；
- 最终总结。

审计记录的目标是让用户知道 AiPanel 做了什么，并能复盘问题。

## 服务器侧边界

AiPanel 默认不在服务器上常驻。

如果未来引入可选 agent，应满足：

- 明确告知用户；
- 可随时卸载；
- 默认不开公网端口；
- 有版本和权限限制；
- 有独立安全说明。

## AI 边界

AI 输出不等于可执行事实。

系统必须假设 AI 可能：

- 生成错误命令；
- 漏掉上下文；
- 误判发行版；
- 误读日志；
- 建议危险操作。

因此必须通过模板、白名单、风险审查和用户确认约束 AI。
