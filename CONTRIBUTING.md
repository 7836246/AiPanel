# Contributing to AiPanel

感谢你愿意参与 AiPanel。

AiPanel 是一个本地运行的 AI 服务器运维客户端，涉及 SSH、命令执行、密钥存储和服务器安全。贡献代码时请优先考虑安全、可审计和可回滚。

## 参与方式

- 提交 bug report；
- 提交功能建议；
- 改进文档、路线图和安全模型；
- 参与 CLI、SSH 执行器、AI 任务规划和桌面客户端开发；
- 补充测试用例和安全边界用例。

## 开发原则

- AI 不应该直接裸跑命令；
- 高风险操作必须有明确风险提示和用户确认；
- 默认优先只读诊断；
- 不把服务器密钥、Token、日志敏感信息写入仓库；
- 命令执行结果需要可审计；
- 尽量让每个任务可以复现和解释。

## 提交 Issue

提交问题时请尽量包含：

- 操作系统和版本；
- 目标服务器系统版本；
- 连接方式，例如密码、密钥、跳板机，注意不要贴出真实凭据；
- 期望行为；
- 实际行为；
- 可复现步骤；
- 已脱敏的日志。

不要在公开 issue 中提交：

- SSH 私钥；
- API Key；
- 服务器公网 IP 和登录密码组合；
- 数据库密码；
- 未脱敏的生产日志。

## 提交 Pull Request

PR 请包含：

- 变更内容；
- 为什么需要这个变更；
- 如何测试；
- 是否涉及命令执行、密钥存储、远程服务器修改；
- 是否新增或改变安全边界。

如果变更涉及高风险操作，例如删除文件、修改防火墙、重启服务、改写系统配置，请在 PR 描述中明确说明风险控制方式。

## Commit 风格

建议使用简短清晰的提交信息：

```text
feat: add ssh connection check
fix: handle docker not installed
docs: add security model
chore: update issue templates
```
